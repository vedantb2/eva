# Eva Launch Twitter Thread

**Tweet 1**
Introducing Eva — an open-source platform that gives AI coding agents full cloud dev environments connected to your repos.

Not a sandboxed LLM. Actual VMs with shell access, package managers, browsers, and the ability to open PRs.

Here's what it does 🧵

---

**Tweet 2**
Quick Tasks — describe what you want built, Eva spins up an isolated cloud sandbox, and an agent executes it end-to-end.

Install deps. Run tests. Build the app. Open a PR.

Run dozens in parallel. Each one gets its own environment.

---

**Tweet 3**
Sessions — persistent cloud dev environments where you and the agent work together in real-time.

Chat, code editor, terminal, and web preview — all in one place. Think pair programming, but your partner never gets tired.

---

**Tweet 4**
Automations — cron-scheduled agent tasks that run against your repo on a cadence.

"Every Monday, audit our API endpoints for missing error handling."
"Every night, check for unused dependencies."

Set it and forget it.

---

**Tweet 5**
Projects — for when a feature is too big for a single task.

Define specs, break them into phases, and let agents execute them step by step. Built-in interview mode helps you refine requirements before any code is written.

---

**Tweet 6**
Design Sessions — iterate on UI with an agent that can actually render what it builds.

Chat-based design refinement with live preview in a real browser. No more "imagine this looks like..." — you see it.

---

**Tweet 7**
Built-in audits — Eva doesn't just ship code, it verifies it.

Automated evaluation of agent output against your requirements. Severity levels, auto-fix suggestions, and activity logs. Trust but verify, on autopilot.

---

**Tweet 8**
The MCP server lets @anthropic's Claude query your @convaborhq or @supabase database directly from claude.ai — no credentials exposed to the model.

Browse schemas, run queries, fetch documents. Your agent finally has context about your actual data.
